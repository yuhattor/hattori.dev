<p>本日、<a href="https://marketplace.visualstudio.com/items?itemName=github.vscode-github-actions">GitHub Actions VS Codeの</a>公式<a href="https://marketplace.visualstudio.com/items?itemName=github.vscode-github-actions">エクステンションの</a>パブリックベータ版のリリースをお知らせします。この拡張機能は、ワークフローのオーサリングと編集をサポートし、IDEを離れることなくワークフローの実行を管理できるようにします。</p>
<p>この拡張機能は、もともとワークフローの実行を監視するための<a href="https://github.com/cschleiden/vscode-github-actions">コミュニティ・プロジェクトとして</a>開始されました。長年にわたってリンチやコード補完の機能を獲得してきましたが、今回のリリースでは公式のGitHub Actionsスキーマを使用するようになったため、最新で最も優れた機能を利用することができるようになりました。また、このリリースの一環として、コミュニティ拡張機能をGitHub Actions製品ファミリーの正式なサポート対象として移行し、GitHubがメンテナンスすることをお知らせします。</p>
<h2 id="managing-workflows-and-monitoring-workflow-runs">ワークフローの管理、ワークフロー実行の監視<a href="#managing-workflows-and-monitoring-workflow-runs" class="heading-link pl-2 text-italic text-bold" aria-label="Managing workflows and monitoring workflow runs"></a></h2>
<p><img decoding="async" src="https://github.blog/wp-content/uploads/2023/03/vscode-actions2.png?w=1024&#038;resize=1024%2C538" alt="" width="1024" height="538" class="aligncenter size-large wp-image-70965 width-fit" srcset="https://github.blog/wp-content/uploads/2023/03/vscode-actions2.png?w=1200 1200w, https://github.blog/wp-content/uploads/2023/03/vscode-actions2.png?w=300 300w, https://github.blog/wp-content/uploads/2023/03/vscode-actions2.png?w=768 768w, https://github.blog/wp-content/uploads/2023/03/vscode-actions2.png?w=1024&#038;resize=1024%2C538 1024w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1" /></p>
<p>この拡張機能を使えば、エディタを離れることなくワークフローを管理することができます。リポジトリ内のワークフローの実行を簡単に監視し、キャンセルして再実行したり、<a href="https://docs.github.com/actions/managing-workflow-runs/manually-running-a-workflow">手動でトリガーしたワークフローを</a>新規にトリガーしたりすることができます。失敗を調査するために、VS Codeを離れることなく、実行からジョブ、ステップへとドリルダウンし、ログを表示することも可能です。</p>
<p>よくあるシナリオは、プッシュされたブランチごとに実行される CI ワークフローです。コードを編集し、それを GitHub にプッシュしてユニットテストと統合テストを実行します。<em>現在のブランチ</em>」ビューでは、VS Codeで現在チェックアウトされているブランチのすべてのワークフローの結果を一目で確認できます。</p>
<p>ワークフローと実行の管理に加えて、組織、リポジトリ、環境というサポートされているすべてのレベルで、利用可能なシークレットと変数を一覧表示することができます。適切な権限があれば、リポジトリや環境の秘密や変数を追加・変更することも可能です。</p>
<p><img decoding="async" loading="lazy" src="https://github.blog/wp-content/uploads/2023/03/image1-7.png?w=1024&#038;resize=1024%2C686" alt="" width="1024" height="686" class="aligncenter size-large wp-image-70942 width-fit" srcset="https://github.blog/wp-content/uploads/2023/03/image1-7.png?w=1999 1999w, https://github.blog/wp-content/uploads/2023/03/image1-7.png?w=300 300w, https://github.blog/wp-content/uploads/2023/03/image1-7.png?w=768 768w, https://github.blog/wp-content/uploads/2023/03/image1-7.png?w=1024&#038;resize=1024%2C686 1024w, https://github.blog/wp-content/uploads/2023/03/image1-7.png?w=1536 1536w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1" /></p>
<h2 id="workflow-authoring">ワークフローのオーサリング<a href="#workflow-authoring" class="heading-link pl-2 text-italic text-bold" aria-label="Workflow authoring"></a></h2>
<p><img decoding="async" loading="lazy" src="https://github.blog/wp-content/uploads/2023/03/vscode-actions3.png?w=1024&#038;resize=1024%2C538" alt="" width="1024" height="538" class="aligncenter size-large wp-image-70966 width-fit" srcset="https://github.blog/wp-content/uploads/2023/03/vscode-actions3.png?w=1200 1200w, https://github.blog/wp-content/uploads/2023/03/vscode-actions3.png?w=300 300w, https://github.blog/wp-content/uploads/2023/03/vscode-actions3.png?w=768 768w, https://github.blog/wp-content/uploads/2023/03/vscode-actions3.png?w=1024&#038;resize=1024%2C538 1024w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1" /></p>
<p>既存の拡張機能に対する改良のほとんどは、ワークフローのオーサリングのために行われました。すべてのエクスペリエンスで公式の GitHub Actions スキーマを使用するようになりました。また、新しい GitHub Actions Expressions パーサーを使用して、ワークフローで式を使用する際のヘルプや洞察をより多く提供します。</p>
<p>ワークフローの編集体験を向上させる機能の一部をご紹介します：</p>
<ul>
<li><strong>ワークフローと式のシンタックスハイライトにより</strong>、実行時に値が挿入される場所が明確になり、複雑な式を理解するのに役立ちます：

</li>
<li>
<p>ワークフロー・スキーマ、式関数、イベント・ペイロードの<strong>統合ドキュメントにより</strong>、必要なコンテキスト・スイッチの数を減らし、ワークフローの編集をスピードアップします：</p>
<p><img decoding="async" loading="lazy" src="https://github.blog/wp-content/uploads/2023/03/image8-1.png?w=1024&#038;resize=1024%2C205" alt="" width="1024" height="205" class="aligncenter size-large wp-image-70945 width-fit" srcset="https://github.blog/wp-content/uploads/2023/03/image8-1.png?w=1999 1999w, https://github.blog/wp-content/uploads/2023/03/image8-1.png?w=300 300w, https://github.blog/wp-content/uploads/2023/03/image8-1.png?w=768 768w, https://github.blog/wp-content/uploads/2023/03/image8-1.png?w=1024&#038;resize=1024%2C205 1024w, https://github.blog/wp-content/uploads/2023/03/image8-1.png?w=1536 1536w" sizes="(max-width: 1000px) 100vw, 1000px" data-recalc-dims="1" /></p>

<li>
<p>YAMLスキーマとGitHub Actions Expressionsの<strong>検証およびコード補完</strong>。ワークフロースキーマ、式関数、イベントペイロード、ジョブまたはステップ出力の検証およびコード補完を即座に実行できます。</p>
<p>この拡張機能は、ワークフローを静的スキーマに対して検証するだけでなく、ワークフローがどのように実行されるかを十分に理解し、ジョブ、ステップ、およびその他のワークフローコンポーネント間の不正確な参照を検出します。また、<code>トリガーを</code>解析し、ワークフローが実行時に受け取る実際のイベントペイロードの検証および補完を行います。例えば、プルリクエストごとにワークフローが実行される場合、拡張機能は<code>github</code> <a href="https://docs.github.com/actions/learn-github-actions/contexts#github-context">コンテキストの</a>イベントペイロードがプルリクエストペイロードであることを認識し、フィールドが存在することを検証してコードの補完を提案することができます。</p>

</li>
<li>
<p>参照されるアクションと再利用可能なワークフローに対する<strong>スマートな検証、コード補完、文書</strong>化：拡張機能は、GitHub.comから参照されるアクションのパラメータ、入力、出力を自動的に解析し、検証、コード補完、インライン文書化を提供します。同じ機能は、ローカルまたは別のリモートリポジトリから参照される<a href="https://docs.github.com/actions/using-workflows/reusing-workflows">再利用可能なワークフローでも</a>機能します。</p>

<h2 id="getting-started">はじめに<a href="#getting-started" class="heading-link pl-2 text-italic text-bold" aria-label="Getting started"></a></h2>
